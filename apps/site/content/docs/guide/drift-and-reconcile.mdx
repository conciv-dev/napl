Once a module's tests pass, its generated files are locked read-only (`0444`). Edit a locked file
anyway and the toolchain treats it as **drift**: `napl gen` refuses to run and `napl status` exits
non-zero (so CI fails). The rule is simple — all change flows through prompts.

## Drift is a compile error with a fix attached

A compile error you can't act on is just a wall, so drift ships with the fix. Before it runs any
agent, `napl gen <target>` checks every module of that target for drift; if in-scope files carry
un-prompted edits it hard-blocks with a guided report per drifted module:

- the module and the exact file(s) that drifted,
- a unified diff of the recorded baseline against your current content (reconstructed by replaying
  the prompt-blame journal; when the baseline predates the journal it falls back to showing the two
  hashes),
- and the three ways out, verbatim:

<Steps>

<Step>

### Fold the edit back into the prompt

`napl reconcile <target> --module <module>` — translate the edit into a proposed prompt amendment,
then leave the module stale for regen.

</Step>

<Step>

### Discard the edit — the prompt wins

`napl gen <target> --module <module> --force` — unlocks, regenerates from the prompt, and re-locks.
`--force` _is_ this resolution: the prompt always wins.

</Step>

<Step>

### Describe the change in the prompt

Edit the prompt to describe what you wanted, then `napl gen <target>`.

</Step>

</Steps>

## Friction at compile time, not edit time

Locks stay `0444` on purpose — not stricter. Editors offer a one-click "make writable", so poking at
generated code to read or experiment stays frictionless. The friction is placed deliberately at
**compile and commit time**, not edit time: `npm run dev`, the test runner, and the bundler never
care about drift. Only `napl gen` and the commit gate confront it.

## Humans edit freely, AI agents can't

The guard rails are asymmetric by design. `napl init` writes, into each generated tree:

- an `AGENTS.md` + `CLAUDE.md` ("this is generated — edit the prompt, not this file"),
- a project `.claude/settings.json` that **denies** the file-editing tools on `.napl/src/**`
  (`Edit(/.napl/src/**)`, which Claude Code applies to Edit/Write/MultiEdit alike),
- a `GENERATED BY NAPL from <prompt> — edit the prompt, not this file` banner atop every source file,
- and a POSIX `pre-commit` hook that runs `napl status` and blocks the commit on drift (WIP escape
  hatch: `git commit --no-verify`).

A human gets the one-click unlock for a quick look; an AI agent that reaches for generated code is
mechanically refused before it types.

## Reconciliation

`napl reconcile` is the sanctioned path for a deliberate edit. It translates the delta against the
generated baseline into a proposed prompt (and test-case) amendment; you approve, the prompt is
updated, and the module is regenerated — regen must reproduce the patched behavior, proven by the
tests. Free-form inverse compilation (guessing a prompt from arbitrary code with no baseline) is out
of scope.
