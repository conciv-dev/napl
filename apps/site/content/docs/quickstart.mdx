---
title: Quickstart
description: Install the binary, write one prompt, generate tested TypeScript, and see the attribution.
---

You need the [Claude Code](https://claude.ai/code) CLI (`claude`) on your `PATH` — `napl gen` reuses your existing login, no API key.

<Steps>

<Step>

### Install

```bash
npm i -g napl-lang
```

One native `napl` binary. Other channels: [Install](/docs/cli/install).

</Step>

<Step>

### Create a project

```bash
napl init
```

Writes `.napl/`, `lock.json`, an example prompt, and the guard rails that keep generated code honest.

</Step>

<Step>

### Write a prompt

<NaplExample module="greeting" filename="greeting.napl" />

Markdown prose plus YAML frontmatter; the tests are data, and they gate the build. This example is editable — delete the `tests:` key and watch the validator object before you ever run the CLI. The second tab is the code it generated.

</Step>

<Step>

### Generate

```bash
napl gen typescript
```

A coding agent writes a real project under `.napl/src/typescript/`, runs its tests until green, maps each sentence to the lines implementing it, and locks the files read-only.

</Step>

<Step>

### Check the connection

```bash
napl status
napl blame .napl/src/typescript/greeting.ts
```

`status` reports `clean`. `blame` shows, per line, the gen and prompt edit that caused it. In VS Code, hover a sentence in `greeting.napl` to see the generated lines that implement it.

</Step>

</Steps>
