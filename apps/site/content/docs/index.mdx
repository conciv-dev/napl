title: What is NAPL
description: The prompt is the source of truth. A coding agent generates per-target code, attributed sentence by sentence and locked read-only.
---

NAPL asks one question: **what if the prompt _was_ the source code?**

You don't write code. You write what you mean, in English, in a `.napl` file:

<NaplExample module="greeting" filename="greeting.napl" />

The example is live — the prompt tab is editable with the CLI's own validator squiggling errors as you type, and the code tab is what `napl gen typescript` generated from it. Try deleting the `module:` line. When you run it for real, the agent:

- scaffolds the project and writes real TypeScript, iterating until the declared tests pass,
- maps every sentence to the exact lines that implement it (hover a sentence above) — if it can't, the build fails,
- locks the result read-only. Edit generated code by hand and `napl status` fails your CI.

The English stays the source of truth. The code is a build artifact: inspectable, tested, traceable, never hand-edited.
